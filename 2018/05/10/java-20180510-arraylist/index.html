ArrayList的fail-fast机制

使用Iterator下，出现fail-fast

ArrayList使用modCount与expectedModCount来进行fail-fast处理。例如，当我们使用iterator进行list的迭代查询时，如果直接添加或者删除一个元素，则会抛出ConcurrentModificationException异常：

ArrayList的iterator()方法返回内部声明的Itr类，其继承结构如下：

查看Itr的代码：

可以看到，当使用迭代器的next()时，首先调用checkForComodification()检测modCount与expectedModCount的值是否相等，而如果在迭代器遍历的时候使用add()方法，则修改了modCount的值，导致modCount != expectedModCount，抛出异常。

而Itr.remove()中通过expectedModCount = modCount;进行同步，解决了该问题，所以在迭代遍历的时候，若需移除元素则需要使用remove方法。
