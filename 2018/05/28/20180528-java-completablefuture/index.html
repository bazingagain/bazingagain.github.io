什么是CompletableFuture?

CompleteableFuture是Java8中提出的异步任务处理API，与Java8中Lambda表达式以及流水线的特性相结合，使得构建异步任务更加简洁直观。

什么场景下使用CompletableFuture？

在互联网中，我们浏览外部网页或使用外部服务的时候，需要与其它外部服务通信，而你并不希望一直等待服务的响应，阻塞当前任务的运行，浪费CPU时钟周期。为了能够充分利用CPU内核，使其处于忙碌状态，我们应避免因等待远程服务的返回，或对数据库的查询，而阻塞线程执行。

结果如下：

执行方式 时间消耗
顺序执行 4120 msecs
并行流 1128 msecs
Completefuture 2091 msecs
Completefuture + Executor 1095 msecs

上面的自定义Executor中，根据最佳线程池大小计算公式：

Nt = Ncpu * Ucpu * (1 + W/C)

其中，Nt为线程池大小，Ncpu为CPU内核数目，Ucpu指CPU利用率（介于0-1之间），W/C为等待时间与计算时间的比率。

其中，W/C为100（因为99%的时间都在等待商店的响应），要使CPU利用率达到100%，本机器为4核CPU，则线程池大小应该计算出来应该为400。但是，我们应当在实际的任务和最高线程数中选取较小的那个值，避免线程浪费。

可以看到，异步方式明显进行快于简单方式，要注意的是异步方式中的几个方法：

  • thenApply() : 同步处理，这里对字符串的解析，没有进行IO操作，可以第一时间完成，所以直接使用同步处理的方式

  • thenCompose() : 将两个异步操作进行流水线， 第一个异步操作完成时，将其返回值作为第二个异步操作的传入参数。
