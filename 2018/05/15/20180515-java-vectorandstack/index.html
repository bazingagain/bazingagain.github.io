Java中的Vector与Stack的类继承结构

Java中，Vector与ArrayList十分相像，底层都是用一个数组作为数据的存储单元，不同的是Vector是同步的(synchronized)，而ArrayList则没有进行同步。Stack继承自Vector，也是同步集合。其类继承结构图如下：

和ArrayList没什么区别，就是add方法添加了 synchronized 修饰符保证多线程访问时的同步。

remove(int index)

removeIf(Predicate<? super E> filter)

removeIf方法接受一个过滤函数，进行数据的移除判断，值得注意的是，其中使用了 BitSet 来记录要移除的数据，BitSet底层使用一个long[] 数组来记录bit位，数组中的每一个元素可以表示64个数，每一个数的存在与否使用0和1进行表示，存在则bit位为1，不存在则bit位为0.参考这篇BitSet的使用中的例子：

bitIndex wordIndex words[wordIndex] words[wordIndex]二进制表示
0 0 1 0001
1 0 3 0011
2 0 7 0111
3 0 15 1111
4 0 31 0001 1111

二进制中的每一个bit的1和0表示对应的数是否出现，如 0001表示0存在，0111表示2，1，0都存在。由于words数组的每个元素只能表示64个数，其对应的数据所对应的部分索引如下：

wordIndex words[wordIndex]数据范围
0 0 ~ 2^6 - 1
1 2^6 ~ 2^7 - 1
2 2^7 ~ 2^8 - 1
3 2^8 ~ 2^9 - 1
… …

这样就形成了一个位图，放到内存当中进行索引查询就非常快。
