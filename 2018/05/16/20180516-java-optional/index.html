Java8中的Optional

Optinal是Java8版本出现的新特性，旨在为了取代null指针，避免null指针所引起的异常行为和Bug。其是一个final类，不能进行进一步的继承。

为什么要使用Optional？

在以前的编码中，我们对对象是否存在一般使用null进行判断，对于比较复杂的逻辑，其判断的深度特别长，例如：

是不是特别麻烦，每一层都要判断一次null，如果一不小心忘了判断则会抛出NullPointerException，充满null判断的代码看起来十分臃肿，Java与C很大的不同就是其一切皆对象，而null指针则是Java中一个独特的存在，破坏了Java的一切皆对象的哲学。而Java8提供了Optional类，可以null指针判断所带来的一系列问题。

Optional的创建

  • Optional car = Optional.empty() 创建一个空Optional
  • Optional car = Optional.of(car) 依据非空值创建一个Optional对象，如果car为null，则立即抛出NullPointerException
  • Optional car = Optional.ofNullable(car) 创建一个运行null值的Optional
