Maven学习之简单应用

学习一项技术最好的方式就是实践，这里，通过创建一个Java应用，对其进行编译、打包等过程，来对Maven的使用进行一个简单的入门。

Maven项目的构建

这里选用通过IDEA来构建Maven项目，直接选用org.apache.maven.archetype:maven-archetype-quickstart创建，设置groupId:com.leon.mavenlearn，artifactId:first,ProjectName:MavenLearn即可得到Maven项目：

使用maven编译项目

编译该项目：

可以看到，mvn clean compile命令首先调用maven-clean-plugin插件删除target下的内容，然后实用maven-resources-plugin插件扫描resources目录，最后使用maven-compiler-plugin插件编译，并输出到target/classes目录下。

使用maven对项目进行测试

对App进行测试，这里我将junit的版本改为了4.12,并将测试用例改为了注解@Test的方式，然后使用mvn clean test命令进行测试：

同样，先对主目录(src/main/java)进行clean、resource、compile操作,然后再使用maven-resources-plugin:2.6:testResources扫描test/resources目录，使用maven-compiler-plugin:3.1:testCompile对test目录下的Java类进行编译并输出到/target/test-classes目录下，最后，使用maven-surefire-plugin:2.12.4:test运行测试

使用maven对项目进行打包和安装

将项目进行打包输出为jar文件，则可以使用maven clean package命令：

值得注意的是maven clean package会进行test步骤，然后才将项目打包到target/目录下。

假如其它项目需要使用该jar，该怎么办呢？这是我们就要将打包好的jar包安装到本地Maven仓库，使用mvn clean install安装：

可以看到，maven-install-plugin:2.4:install将first-1.0-SNAPSHOT.jar安装到了本地的/Users/alex/.m2/repository目录下。

另外，这种方式也可打包成可执行的jar包：

总结

  • mvn clean compile 编译项目
  • mvn clean test 编译并运行测试代码
  • mvn clean package 编译、运行测试、将项目打包成jar包
  • mvn clean install 编译、运行测试、将项目打包成jar包、并将jar包安装到本地repository

对于打包可执行jar包，可以使用maven-shade-plugin或maven-jar-plugin等插件
