为什么要使用Maven？

在以前刚学习使用Java的时候，对于要使用的第三方Jar包，都去网上到处搜集，每次引用的变更，都需要在IDE中进行繁复的配置工作，在实习的时候，公司使用Maven来管理Java项目，记得当时需要进行某个依赖包的升级，而只需更改dependency的version的版本号就立马实现了，当然可能存在依赖冲突。

Maven是Apache下的一个项目，是一个强大的软件工程管理工具，它基于POM（project object model 项目对象模型）的概念而构建。通过Maven，你能够快速地搭建项目，它提供很好的解耦能力。

Maven安装目录的结构

其中，bin目录主要存放一些运行命令，boot目录下的plexus-classworlds-2.5.2.jar是一个类加载器，Maven使用这个加载器加载自己的类库，最重要的是conf下的settings.xml文件，如果直接在安装目录下更改它，则控制的是整个maven，一般我们把setting.xml文件复制到 ~/.m2/下，然后对其进行修改，则控制的是这个用户的Maven的行为, ~/.m2/repository下则存放的是下载的构建库，Lib目录下则存放Maven运行时所需要的Java类库。
